- [x] 6.x.2 **Bug 2 ŌĆö `excRate` reference in `transaction_filter_set.dart` lines 188ŌĆō194** (Phase 9 + Phase 6 risk #1). Type: cross-query SQL alias coupling. Root cause: `TransactionFilterSet.toTransactionExpression()` is shared by both `getTransactionsWithFullData` (which has the `excRate` JOIN) and `countTransactions` (which Phase 9 dropped the JOIN from); the `minValue` / `maxValue` predicates referenced `excRate.exchangeRate` and would crash any saved filter that flowed through the count path.
